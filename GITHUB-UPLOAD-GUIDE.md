# העלאת אתר תורת אבי ל-GitHub

## מה עושים בכל עדכון

1. מסיימים את העדכון באתר במחשב.
2. בודקים שהאתר נראה טוב.
3. מכינים תיקייה נקייה להעלאה.
4. מעלים את התיקייה ל-GitHub.
5. GitHub Pages מציג את האתר המעודכן.

## בפעם הראשונה בלבד

1. נכנסים ל-GitHub ופותחים חשבון.
2. לוחצים על New repository.
3. נותנים שם, למשל `torat-avi-site`.
4. משאירים Public.
5. יוצרים את המאגר.
6. מעלים את קבצי האתר.
7. נכנסים ל-Settings.
8. נכנסים ל-Pages.
9. בוחרים Deploy from a branch.
10. בוחרים branch בשם `main` ותיקייה `/root`.
11. לוחצים Save.

אחרי כמה דקות GitHub ייתן קישור לאתר.

## מה חשוב לא להעלות

לא מעלים תיקיות עבודה כמו:

- `node_modules`
- `.wrangler`
- `functions`
- `migrations`
- `reports`
- `tools`

לכן עדיף בכל פעם להשתמש בתיקייה הנקייה שאני מכין להעלאה, ולא לגרור את כל תיקיית העבודה.

## משפט קצר לזכור

GitHub הוא המחסן הקבוע של האתר.  
GitHub Pages הוא מה שמציג את האתר לגולשים.
